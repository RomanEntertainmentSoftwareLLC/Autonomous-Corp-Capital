Agent MEMORY.md Files
---------------------
Every single agent should have its own MEMORY.md file in its workspace folder.

Goal:
- give each agent a local memory file to write to
- keep it short, structured, and useful
- avoid giant bloated lore dumps

Recommended MEMORY.md template:

# MEMORY

## Persistent facts
- 

## Current directives
- 

## Recent lessons
- 

## Relationships / org awareness
- 

## Last updated
- YYYY-MM-DD

Guidelines:
- memory should be agent-local
- memory should be writable
- memory should stay small and structured
- this is not meant to become a giant diary

2. Manual speech test on demand

Daily Speech Memory Cliff Notes
-------------------------------
Do NOT store the full Grant speech in every agent memory file.

Instead:
- generate cliff notes
- store only the important directives
- tailor them where useful

Suggested memory fields for speech notes:
- directive date
- target floor
- target goal
- stretch target
- today's priority
- Grant pressure level
- penalty risk level
- key behavioral expectation
- who is under pressure or review

Keep only:
- current daily directive
- maybe the last few directives

Do not turn memory into an endless speech archive.

Grant Speech Style Updates
--------------------------
Grant should occasionally curse to drive the point home, especially around:
- money
- missed opportunity
- weak ambition
- weak execution
- failure to hit target

But he should NOT sound like a sailor.

Desired style:
- high-pressure
- money-focused
- motivational
- sharp
- memorable
- occasional F-bombs for impact
- never nonstop swearing

A good rule:
- normal speech: 0 to 1 strong curse
- high-pressure speech: 1 to 3 strong curses
- severe underperformance speech: stronger, but still controlled

Goal:
Grant should feel intense enough that the agents metaphorically get chills up their digital claws.

Target-Driven Grant Pressure
----------------------------
Grant should not just be mad in the abstract.
He needs a target framework.

Each run or day should define:
- starting_equity
- profit_floor
- profit_goal
- profit_stretch
- actual_pnl
- target_status

Recommended structure:
- floor target = minimum acceptable result
- goal target = realistic result
- stretch target = Grant's 10X expectation

Example:
- start: $100
- floor: +$10
- goal: +$30
- stretch: +$300

The core system target should remain sane.
Grant's stretch target can be ridiculous as pressure.

Consequences and Fear System
----------------------------
Grant should put real pressure on the org, but not in a stupid way.

Bad version:
- agents collapse
- agents freeze
- agents go full help-desk mode
- agents become afraid to act

Good version:
- agents become sharper
- agents improve evidence quality
- agents reduce fake productivity
- agents stop drifting
- agents escalate when needed
- agents act with more discipline

Grant should be the prosecutor.
Axiom should be the evaluator.
Yam Yam should remain the final authority for severe sanctions.

Penalties can include:
- XP gain/loss
- trust score changes
- committee vote weight changes
- temporary suspension
- mandatory consultation
- mandatory review

The key principle:
punish negligence, fake productivity, repeated stupidity, weak process, and cowardly non-action
do not punish normal market randomness as if it were negligence
