# 2026-04-11

- Paper-proof runs need run-local bridge/ledger proof artifacts; global history alone is not enough to prove low bridge usage for a specific run.
- Ledger token/cost evidence should be surfaced only when the data already exists in the call path; never fabricate pricing or costs.
- Paper-run stop/finalizer paths should emit run-local proof artifacts before clearing the current run pointer, or aborted runs can lose their local evidence.
- If a task starts dragging or bouncing through aborts, split it into smaller proof/verify steps instead of pushing one big all-at-once run.
- Robinhood auth clients should mirror ACC exactly: load secrets with `tradebot.secrets.load_secrets()`, sign `api_key + timestamp + path + method + body`, and derive `SigningKey` from the first 32 bytes of the decoded private key; `market-comparison.py` imports `nacl.signing` at module scope, so it can fail to import when `pynacl` is missing.
- Step 79 proof should be classified into three separate gates: core paper-mode/runtime safety, bridge execution proof, and OpenAI token/cost proof; token/cost is a separate enhancement unless a real OpenAIAdapter-backed path is actually exercised.
- `live_run.py` should keep paper as the default and require an explicit `--live-trade` flag before any live trading path can run.
- Replace `datetime.utcnow()` in `tools/live_run.py` with timezone-aware UTC datetimes so the runtime avoids deprecation warnings without changing behavior.
