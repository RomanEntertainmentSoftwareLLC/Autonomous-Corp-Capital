# RPG STEP 18: MANUAL RUNTIME SCORE APPLICATION PLAN

## INPUTS REQUIRED

Before applying a score, the following must be documented:

| Input | Source | Required |
|-------|--------|----------|
| Run ID | Operator assigns | YES |
| Date | ISO8601 timestamp | YES |
| Agent Name | Who performed the work | YES |
| Verified Outcomes | What actually happened | YES |
| XP Calculation | Per RPG_RULES.md formulas | YES |
| Proof References | File paths, log entries | YES |
| Reviewer Name | Who approves | YES |

### Example Input Set
```
Run ID: run_2026-03-29_001
Date: 2026-03-29T19:00:00Z
Agent: yam_yam
Verified Outcomes:
  - Created RPG_RULES.md (proof: file exists)
  - Re-read verification performed (proof: read result)
XP Calculation:
  - Create file: +5 REL
  - Re-read: +2 REL
  - Subtotal: +7 REL
  - First-try: 1.5x multiplier
  - Total: 7 × 1.5 = 10.5 → 11 XP (+11 REL)
Proof References:
  - ai_agents_memory/yam_yam/RPG_RULES.md
  - ai_agents_memory/yam_yam/CHANGE_HISTORY.md
Reviewer: Jacob
```

## WHO REVIEWS

### Primary Reviewer: Operator (Human)
- Verifies outcomes actually occurred
- Checks proof references exist
- Validates XP calculation math
- Approves or rejects the score

### Secondary Review: Agent Self-Report
- Agent proposes XP based on their actions
- Agent provides proof references
- Agent does NOT apply their own score

### Escalation
- If disputed: Review CHANGE_HISTORY.md, TASK_HISTORY.md
- If unclear: Request additional proof
- If gaming suspected: Penalty per RPG_RULES.md

## WHICH FILES CHANGE

### 1. Run Score File (NEW)
**Path:** `rpg_runs/{run_id}.md`

Created from template for each run. Contains:
- Run metadata
- Agent score entries
- Operator approval

### 2. Agent RPG_HISTORY.md (APPEND)
**Path:** `ai_agents_memory/{agent}/RPG_HISTORY.md`

Appended with session summary:
```markdown
## Session {run_id}

| Event | XP | Stat | Proof |
|-------|-----|------|-------|
| Created RPG_RULES.md | +5 | REL | file exists |
| Re-read verification | +2 | REL | read result |

**Session Total: +10 XP**
**Approved By: {reviewer}**
```

### 3. Agent RPG_STATE.md (MODIFY)
**Path:** `ai_agents_memory/{agent}/RPG_STATE.md`

Updated with new totals:
- Total XP increased
- Stat values recalculated
- Level checked (and updated if threshold crossed)
- Sessions count incremented

## HOW XP IS APPLIED

### Step 1: Calculate Gross XP
Apply RPG_RULES.md formulas to verified outcomes:
```
Base XP: +7 (create file + re-read)
Multiplier: 1.5x (first-try)
Gross XP: 10.5 → round to 11
```

### Step 2: Apply Penalties
Subtract any documented failures:
```
Gross XP: +11
Penalties: -3 (tool failure earlier)
Net XP: +8
```

### Step 3: Distribute to Stats
Per RPG_RULES.md, assign to appropriate stat:
```
+8 REL (file operations)
```

### Step 4: Check Caps
- Session XP cap: 200 max
- Anti-death-spiral: max -3 per stat per session

---

## HOW RPG_HISTORY IS UPDATED

### Append-Only Rule
Never modify existing entries. Always append new session data.

### Format
```markdown
## Session {run_id} - {date}

| Time | Event | XP | Stat | Proof |
|------|-------|-----|------|-------|
| 19:05 | Created file | +5 | REL | path/to/file |
| 19:06 | Re-read verify | +2 | REL | read result |

**Session Net: +7 XP**
**Distribution: +7 REL**
**Approved By: {reviewer}**
**Approved At: {timestamp}**
```

### What NOT to Include
- Subjective commentary
- Unverified claims
- Planned work (only completed work)
- Chatter or conversation

---

## HOW RPG_STATE IS UPDATED

### Read Current State
```markdown
| Field | Value |
|-------|-------|
| Total XP | 45 |
| Reliability | 10 |
| Sessions | 2 |
```

### Calculate New State
```
New Total XP: 45 + 7 = 52
New REL: floor(52 REL-XP / 10) = 5
New Sessions: 2 + 1 = 3
```

### Check Level Advancement
```
Current Level: 1 (0 XP required)
Next Level: 2 (100 XP required)
Progress: 52/100 (52% to Level 2)
```

### Write Updated State
Replace values in RPG_STATE.md table. Keep format identical.
