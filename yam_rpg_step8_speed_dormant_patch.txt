## WHY SPEED IS DORMANT

### Technical Reason
Speed requires timing data that does not currently exist in the runtime:
- No task start timestamps captured
- No task end timestamps captured
- No duration calculations available
- No task complexity baseline established

### Data Gap
As documented in yam_rpg_step4_signal_map.txt:
| Stat | Current Evidence | Status |
|------|------------------|--------|
| ACC | Strong (CHANGE_HISTORY.md, MISTAKES_TO_AVOID.md) | Active |
| REL | Strong (TASK_HISTORY.md, tool results) | Active |
| JUD | Medium (CHANGE_HISTORY.md risk notes, AGENTS.md) | Active |
| SPD | None | Dormant |

### Activation Criteria
Speed will activate when:
1. TASK_HISTORY.md includes `timestamp_start` and `timestamp_end` fields
2. Agents consistently log task durations
3. Minimum 100 timed tasks exist in dataset
4. Operator explicitly enables Speed scoring
